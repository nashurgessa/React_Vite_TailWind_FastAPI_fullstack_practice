This is for students' practice project.

# Target
We would not specify exact features that everyone must implement. Otherwise, every project becomes the same.
Instead, We would define feature levels. Students choose features according to their own project, interests, and difficulty level.


# Topics


| Day        | Main Topic     | New Knowledge                                           |
| ---------- | -------------- | ------------------------------------------------------- |

| Day 12     | Pydantic       | Validate input/output                                   |

| Day 14     | Advanced APIs  | Search, pagination, sorting, dependency injection, CORS |

# System Features

| **Level**                           | **Description**                                                                                                                                | **Examples**                                                                                                                                                                                  |
| ----------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Core (Required)**                 | Fundamental features that every complete information system should include.                                                                    | Login/Register, CRUD, MySQL database, Search, Filtering, Sorting, Pagination, Input Validation, Error Handling                                                                                |
| **Intermediate (Choose 4–6)**       | Practical features that improve usability and demonstrate software engineering skills. Students choose those appropriate for their project.    | Image/File Upload, Dashboard, Statistics, Export (CSV/PDF), Search Suggestions, Multi-condition Search, Role-Based Access Control, Activity Log, Booking/Purchase History, Email Notification |
| **Advanced (Choose 2+)**            | Challenging features that showcase independent learning and creativity. Suitable for stronger projects, portfolios, and thesis demonstrations. | AI Chatbot, AI Recommendation System, QR Code, Interactive Maps, Calendar/Scheduling, Real-time Notification (WebSocket), Data Visualization, Dark Mode, Responsive Design                    |
| **Software Engineering (Required)** | Demonstrates professional development practices rather than user-facing functionality.                                                         | Git Version Control, README, Swagger/OpenAPI Documentation, ER Diagram, System Architecture Diagram, Unit/API Testing, Logging, Environment Configuration (.env), Modular Project Structure   |
